Design Diary

This design diary was an assignment for my Understanding Users class with the following guidelines: Create an observational diary of user interactions with any information environment (computer, person, facility) where the interactive experience is challenged by poor or uninformed design choices. These interactions can be personally experienced or observed in others. Each entry should cover the context of occurrence, the nature of the problem from the userâ€™s perspective, an analytical/theoretical framing of the experience, and a clear recommendation for re-design that would improve the experience. Cite appropriate literature for each entry.

Personal Goals:

  1. Challenge my communication skills - Developing the ability to communicate effectively in a short span of time (or small amount of space) is important in our increasingly complicated world. My goal was to communicate my message using only a single page without sacrificing context or content. 
  2. Become more familiar with Sketch - This project gave me the opportunity to improve my sketch skills and do so in a repetitive way that I believe allowed me to become more familiar with the wide array of tools that Sketch offers. I wanted to create clean designs but more importantly I wanted to direct the users attention appropriately.
